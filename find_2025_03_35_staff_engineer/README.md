# 事業を差別化する技術を生み出す技術
## はじめに
近年、エンジニアの評価制度やキャリアパスの整備が進み、特に技術領域で卓越した貢献を果たすエンジニアに対する期待が高まっている。GMOペパボでは、エンジニアのキャリアパスとして、エンジニアリングマネージャーのラダーとエンジニアリングスペシャリストのラダーを明確に分け、それぞれの役割に応じた評価制度を整備している。本稿では、エンジニアリングスペシャリストとしての役割を踏まえながら、事業を差別化する技術をどのように生み出し、それを組織全体で活用していくかについて考察する。

### エンジニアリングスペシャリストの役割

エンジニアリングスペシャリストは、単なる技術者ではなく、企業や事業の方向性を技術的な観点から支え、推進する役割を担う。特に、GMOペパボでは、CTOや事業部のリーダーと協力しながら、未来の事業成長に必要な技術を導入し、組織に定着させることが求められている。この役割を果たすためには、技術力だけでなく、事業の本質を理解し、適切な技術選定と推進を行う能力が不可欠である。

### 技術基盤チームの役割

技術基盤チームは、原則プリンシパルエンジニア以上のエンジニアが所属する、事業全体の技術的な基盤を支える役割を担っている。このチームでは、サービス開発チームが直面する技術的課題を解決し、スケーラビリティや安定性を向上させるためのアーキテクチャ設計やインフラ整備を行ったり、事業の課題になっている問題を解決することが求められる。 加えて、新しい技術の検証や導入も積極的に進め、事業の競争力を技術面から強化することを使命としている。

## 事業を差別化する技術を生み出す技術

### 事業を差別化する技術とは

技術の導入は、単に最新のトレンドを追うことが目的ではない。事業の特性を理解し、それに適した技術を選定・実装することが重要である。GMOペパボにおいては、**「オンプレミス」と「マルチテナント」** という二つのキーワードが技術戦略の中心にある。例えばIaaSやSaaSを利用する際にドメイン単位での課金というモデルがそれなりにあり、そういった場合においては外部に支払う代金が原価を高騰させる要因になる。故に、自社で技術開発を行い、コストを落とすことで競争力を高めることが重要である。

### 具体的な技術事例

GMOペパボでは、事業を差別化する技術として、以下のような事例がある。

**STNS**（Simple Toml Name Service）: オンプレミスサーバーの権限管理の効率化を目的として開発された認証基盤。従来のLDAPやMySQL に代わるシンプルな仕組みを提供し、全社のサーバの認証における運用負荷を軽減。

**仮想待合室**: 高負荷時にユーザーを適切に制御することで、サービスの安定性を確保。特に、キャンペーン時や大量アクセスが発生するシチュエーションで効果を発揮。

**内製Kubernetes Engine**: クラウドに依存せず、自社のオンプレミス環境に最適化したKubernetes基盤を構築。コンテナ化のメリットを活かしながら、ペパボの複数事業、マルチテナント環境に適した運用を実現。

これらの技術は、事業の特徴を理解し、それに最適化された形で設計・運用されている点が共通している。単なる技術の導入ではなく、事業戦略に適合した形で技術を活用することが、差別化につながる要素となる。

## メンバーとの協業

### 個人の限界とチームの力

技術的な課題の解決には、個々のエンジニアの努力だけでなく、チームとしての協力が不可欠である。高度な技術開発や、事業全体に影響を与えるアーキテクチャの決定には、多様な視点や知見が求められる。特に、事業を差別化する技術を生み出すためには、エンジニア同士の協業を積極的に進めることが重要である。

### 協業の方法

私は、エンジニアの成長と知見の共有を促進するために、**シャドーイング** と **逆シャドーイング** という手法を活用している。

**シャドーイング**: 自身が作業するときに行動に至るまでの思考や、切り分け方を他のメンバーに見てもらい、学習を促す。

**逆シャドーイング**: 他のメンバーに適当な粒度でタスクを実行してもらい、適宜フィードバックを行いながら、技術的な理解を深める。

※  スタッフエンジニアの道ではこれらの用語を同期的な作業に対して使っているが、私は非同期な作業に対して使っている。

このような取り組みを通じて、エンジニア間の技術的な相互理解が深まり、新たな技術の導入、理解や事業の差別化につながるアイデアが生まれる。

## まとめ

事業を差別化する技術を生み出すためには、以下の要素が重要である。

**事業の特色を理解すること**: 技術は目的ではなく、事業の成長や競争力強化の手段である。事業の特性を踏まえた技術選定が不可欠。

**技術の研鑽と楽しむ姿勢**: 日々の技術的な挑戦を楽しみながら、新たなアイデアを生み出すことが重要。

**仲間との協力**: 一人でできることには限界がある。チームで協力しながら技術的な課題を解決し、事業価値を高めることが求められる。

本稿で紹介した事例やアプローチを参考に、自社に適した技術戦略を考え、事業の競争力向上につなげてほしい。
